# iOS Classic AI PWA ## Product Overview **The Pitch:** A native-feeling, iOS-styled Progressive Web App for deep AI character interactions. It brings desktop-grade LLM configuration and lore management (Worldbook) into a frictionless, familiar mobile interface. **For:** Power users of local and API-based LLMs who want to manage complex roleplay and lore on the go without sacrificing the native iOS feel. **Device:** mobile **Design Direction:** Unapologetically Apple. Strict adherence to iOS Human Interface Guidelines with absolute minimalism, heavy use of `backdrop-filter: blur`, San Francisco typography, and grouped list styles. **Inspired by:** Apple Messages, iOS Settings, SillyTavern --- ## Screens - **Chats:** Grouped list of recent conversations with characters - **Chat Detail:** The active conversation interface with message bubbles and sticky input - **World Info:** Master list of lore entries and character definitions - **Entry Editor:** Detail view to create/edit a specific lore key and its content - **Settings:** Grouped list for app preferences and navigation to API configs - **API Config:** Form to input API keys, select endpoints, and adjust LLM parameters --- ## Key Flows **Configure LLM API:** Connect the app to an external provider 1. User is on Settings -> sees API Connections list 2. User clicks OpenAI/Custom Endpoint -> opens API Config screen 3. User enters URL and Key, taps "Test Connection" -> Success badge appears **Add World Lore:** Inject contextual facts for the AI 1. User is on World Info -> sees list of existing lore 2. User clicks "+" in top navigation -> opens Entry Editor 3. User enters keywords and content, taps "Save" -> Lore is active for next chat ---
Design System ## Color Palette - **Primary:** `#007AFF` - iOS System Blue for interactive elements, CTAs - **Background:** `#F2F2F7` - iOS grouped background color - **Surface:** `#FFFFFF` - Cards, list items, chat bubbles (receive) - **Text:** `#000000` - Primary labels, body text - **Muted:** `#8E8E93` - iOS System Gray for subtitles, placeholders - **Accent:** `#34C759` - iOS System Green for success states, active toggles ## Typography Strict adherence to Apple's San Francisco typeface to achieve the native illusion. - **Headings:** SF Pro Display, 700, 34px (Large Title), 17px (Inline Nav Title) - **Body:** SF Pro Text, 400, 17px (Standard iOS body) - **Small text:** SF Pro Text, 400, 13px (Footnotes, timestamps) - **Buttons:** SF Pro Text, 600, 17px **Style notes:** Heavy use of `backdrop-filter: blur(20px)` with `rgba(255, 255, 255, 0.7)` for navigation and tab bars. Standard 10px radius for list groups, 18px for chat bubbles. Hairline borders (`0.5px solid #C6C6C8`) for dividers. ## Design Tokens ```css :root { --color-primary: #007AFF; --color-background: #F2F2F7; --color-surface: #FFFFFF; --color-text: #000000; --color-muted: #8E8E93; --color-accent: #34C759; --color-border: #C6C6C8; --color-nav-bg: rgba(255, 255, 255, 0.75); --font-primary: 'SF Pro Display', 'SF Pro Text', -apple-system, BlinkMacSystemFont, sans-serif; --radius-group: 10px; --radius-bubble: 18px; --spacing-edge: 16px; --spacing-list: 20px; } ```

Screen Specifications ### Chats **Purpose:** Landing screen to view and resume recent conversations **Layout:** Standard iOS large title navigation top, grouped list middle, bottom tab bar. **Key Elements:** - **Navigation Bar:** 44px height (excluding status bar), blurred background, "Edit" left, compose icon right. 34px "Chats" large title below. - **Chat Cell:** 76px height. 45px circular avatar left, user name 17px bold top-right of avatar, preview text 15px muted below name. Time indicator 15px muted far right. - **Tab Bar:** 83px height, blurred, icons for Chats, World, Settings. **States:** - **Empty:** "No Chats" 20px gray text centered, "Start a new conversation" 15px button below. **Interactions:** - **Swipe Left on Cell:** Reveals red "Delete" action button (classic iOS swipe-to-delete). ### Chat Detail **Purpose:** Active conversation interface **Layout:** Inline navigation top (avatar and name), scrolling messages middle, sticky input area bottom. **Key Elements:** - **Header:** Blurred, 17px bold character name centered, `< Chats` back button left. - **User Bubble:** `#007AFF` background, `#FFFFFF` text, right-aligned, 18px radius (2px bottom-right radius). - **AI Bubble:** `#E9E9EB` background, `#000000` text, left-aligned, 18px radius (2px bottom-left radius). - **Message Input:** 36px height input field, grey pill shape, expanding up to 120px height. Blue up-arrow submit button inside right edge. **Interactions:** - **Scroll up:** Keyboard dismisses, header transitions from transparent to blurred. ### World Info **Purpose:** Manage lore dictionary **Layout:** Searchable grouped list **Key Elements:** - **Search Bar:** 36px height, `#767680` background at 12% opacity, 10px radius, magnifying glass left, "Search" placeholder. - **Entry List:** Grouped style (`#FFFFFF` block, 10px radius). Each row has 17px bold keyword, 15px muted preview of lore. 0.5px divider between rows. - **Add Button:** "+" icon top right in nav bar, `#007AFF`. **States:** - **Empty Search:** "No Results" text centered in remaining space. ### Entry Editor **Purpose:** Create or modify a World Info entry **Layout:** Form fields styled as iOS grouped lists **Key Elements:** - **Header:** "Cancel" left, "New Entry" center, "Done" right (bold). - **Keywords Field:** Standard list row, "Keywords" label left, text input right. - **Content Area:** 200px height text area, `#FFFFFF` background, 10px radius, 16px padding. - **Insertion Strategy:** Segmented control (Before, After, System) to dictate where lore injects into context. **Interactions:** - **Tap Done:** Validates fields, returns to World Info list, shows brief success toast. ### Settings **Purpose:** Root configuration menu **Layout:** iOS Settings grouped layout **Key Elements:** - **User Profile:** 80px high cell, large avatar, generic user settings. - **Connection Group:** Grouped list containing "API Endpoints", "Context Size", "Generation Params". - **Toggle Row:** "Stream Responses" label left, native green toggle right. ### API Config **Purpose:** LLM connection details **Layout:** Grouped input forms **Key Elements:** - **API URL:** Full width input, `https://...` placeholder. - **API Key:** Password masked input, "sk-..." placeholder. - **Test Button:** Full width list item, `#007AFF` text centered. - **Status Indicator:** 12px dot (green for connected, red for error) next to "Connection Status" label. **States:** - **Loading:** Test button changes to standard iOS gray spinner. - **Error:** Red text appears below form explaining API rejection.

Build Guide **Stack:** HTML + Tailwind CSS v3 + JavaScript (Vanilla/Alpine for state) **Build Order:** 1. **Settings / API Config:** Establish the foundational iOS grouped list styling, background colors, and form inputs. 2. **Chats / World Info:** Reuses the list styling but introduces the Large Title navigation and bottom tab bar structural shell. 3. **Chat Detail:** The most complex screen. Requires flexbox scrolling, sticky bottom input that handles keyboard appearance, and message bubble styling. 4. **Entry Editor:** Finalizes form inputs with text areas and segmented controls.
