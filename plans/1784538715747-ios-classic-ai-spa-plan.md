# iOS Classic AI PWA - SPA Implementation Plan

## Architecture Decision

**Single-Page Application (SPA)** with hash-based routing and a phone-style home screen launcher.

- Eliminates iframe/page-reload flickering and crashes
- All sub-apps share one IndexedDB, one Service Worker, one PWA install
- New apps = new JS module + one route entry
- Lazy loading for sub-app JS modules

---

## HTML Structure (index.html)

Single HTML file with:
- CDN links: Tailwind CSS, Be Vietnam Pro font, Material Symbols, idb library
- `<div id="app"></div>` — all page content rendered here
- `<script type="module">` loading `js/app.js`

No page-specific HTML. All views are generated by JS page modules.

## JS Architecture

### router.js
- Hash-based routing: `#/home`, `#/chats`, `#/chat/:id`, `#/world-info`, `#/entry-editor/:id`, `#/settings`, `#/api-config`
- `router.on(pattern, handler)` — register route
- `router.navigate(path)` — programmatic navigation
- `router.start()` — listen to `hashchange` event
- Route params extraction (e.g. `:id`)
- Before-leave hooks (for unsaved form data)

### db.js
- Initialize IndexedDB `ios-classic-ai` with stores: chats, messages, worldInfo, characters, settings
- CRUD operations for each store
- Uses idb library (CDN)

### api.js
- OpenAI-compatible API client
- Streaming via `fetch` + `ReadableStream`
- World Info context injection (keyword matching → prepend/append to prompt)
- Error handling with retry
- Connection test endpoint

### components.js
Factory functions returning DOM elements:
- `createIOSNavBar(options)` — large title or inline
- `createIOSGroupedList(sections)` — grouped card list
- `createIOSListCell(options)` — single row cell
- `createIOSSearchBar(onSearch)` — search input
- `createIOSSegmentedControl(segments, onChange)` — tab selector
- `createIOSToggle(checked, onChange)` — switch
- `createIOSSlider(min, max, step, value, onChange)` — range slider
- `createKakaoBottomNav(tabs, activeTab, onTabChange)` — bottom tab bar
- `createKakaoFAB(onClick)` — floating action button
- `createKakaoChatCell(options)` — chat list item
- `createKakaoBubble(role, content, avatar, name)` — chat message bubble
- `createToast(message, type)` — feedback toast

### app.js
- Initialize router, database, API client
- Register all routes with lazy-loaded page modules
- Global state management (current chat, settings cache)
- Home screen launcher (app icon grid)
- PWA install prompt handling
- Service worker registration

### pages/home.js
- Phone-style home screen with app icons in a grid
- Icons: Chat, World Info, Settings
- Each icon navigates to its sub-app route
- iOS-style wallpaper background

### pages/chats.js
- Kakao-style chat list
- Load chats from IndexedDB, sorted by last_updated
- Search filtering
- FAB → new chat (character selection)
- Bottom nav: Friends, Chats (active), Explore, More
- Tap cell → navigate to `#/chat/{id}`

### pages/chat.js
- Kakao-style chat detail
- Load messages from IndexedDB
- Render bubbles with tails
- Expanding textarea input
- Send → save user message → API call with World Info → stream AI response
- Auto-scroll to bottom
- Back button → `#/chats`

### pages/world-info.js
- iOS-style grouped list
- Load entries from IndexedDB
- Search by keywords
- Tap cell → `#/entry-editor/{id}`
- "+" button → `#/entry-editor` (new)
- Bottom nav: Chats, World Info (active), Settings

### pages/entry-editor.js
- iOS-style form
- Keywords input with chip display
- Content textarea (200px)
- Insertion segmented control (Before, After, System)
- Priority number input
- Enabled toggle
- Cancel → back, Done → validate & save → back with toast

### pages/settings.js
- iOS-style grouped sections
- Profile card, Account, App Settings, Intelligence, About
- Tap "API 設定" → `#/api-config`
- Bottom nav: Chats, World Info, Settings (active)

### pages/api-config.js
- iOS-style form
- API URL, API Key (password), Context Size
- Test Connection button with status indicator
- Generation parameters: Temperature, Top P, Frequency Penalty, Presence Penalty
- Save to IndexedDB settings store

## Key Flows

### Send Message
1. User types in chat input → send button activates (Kakao Yellow)
2. Tap send → save user message to IndexedDB
3. Optimistic render: user bubble appears immediately
4. Build API request: system prompt + World Info (keyword match) + message history
5. Stream API response → AI bubble updates in real-time
6. On complete → save AI message to IndexedDB

### Add World Lore
1. World Info page → tap "+" → navigate to `#/entry-editor`
2. Fill keywords (chip input), content, insertion, priority, enabled
3. Tap "Done" → validate → save to IndexedDB → navigate back → show toast

### Configure API
1. Settings → tap "API 設定" → navigate to `#/api-config`
2. Enter URL, Key → tap "Test Connection"
3. API client sends test request → green dot on success, red on failure
4. Adjust generation parameters → auto-save on change

## PWA Configuration

### manifest.json
- `start_url`: `/index.html`
- `display`: `standalone`
- `theme_color`: `#007AFF`
- `background_color`: `#F2F2F7`
- `orientation`: `portrait`

### sw.js
- Cache-first strategy for static assets (HTML, CSS, JS, fonts)
- Network-first for API calls
- Cache versioning for updates

## iOS-Specific Considerations

- `env(safe-area-inset-*)` for notch/dynamic island
- `-webkit-overflow-scrolling: touch` for smooth scrolling
- `backdrop-filter: blur(20px)` with `-webkit-backdrop-filter` fallback
- 0.5px hairline borders via `transform: scaleY(0.5)`
- `standalone` display mode hides Safari chrome
- 7-day IndexedDB cleanup risk: implement export/backup feature in settings

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| iOS 7-day IndexedDB cleanup | Add export/import backup in settings; prompt user to open app weekly |
| Memory pressure on older iOS | Lazy load page modules; destroy DOM on route leave |
| Tailwind CDN size | Consider switching to Tailwind CLI build for production |
| Streaming API on Safari | Use `ReadableStream` polyfill if needed; fallback to polling |
| iOS standalone mode back gesture | Implement custom back button in all nav bars |
